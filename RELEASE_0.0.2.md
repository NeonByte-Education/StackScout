# v0.0.2: Базовые CRUD-операции и бизнес-логика

Второй релиз StackScout знаменует переход от инфраструктурного каркаса к функциональной системе. В этой версии реализованы основные операции работы с данными и ключевые сущности платформы, что превращает проект из "скелета" в полноценное работающее приложение с REST API.

### Что сделано в этом выпуске:

- **Модели данных**: Реализованы три ключевые сущности - Library (библиотеки), License (лицензии) и ScanJob (задачи сканирования) с полной JPA-разметкой и индексами для оптимизации запросов
- **CRUD операции**: Разработаны полнофункциональные REST контроллеры для всех сущностей с поддержкой создания, чтения, обновления и удаления данных
- **Слой репозиториев**: Настроены Spring Data JPA репозитории с кастомными методами поиска и фильтрации (поиск по имени, источнику, статусу, типу лицензии)
- **Бизнес-логика**: Реализованы сервисы с продвинутой функциональностью - пагинация, сортировка, поиск библиотек, фильтрация по "здоровью", статистика по лицензиям и задачам
- **API документация**: Все эндпоинты аннотированы через Swagger/OpenAPI с подробными описаниями операций и параметров
- **Валидация данных**: Интегрирована Bean Validation для проверки входных данных на уровне DTO
- **Обработка ошибок**: Настроен глобальный обработчик исключений с понятными сообщениями об ошибках

### Реализованные эндпоинты:

**Библиотеки (/api/v1/libraries)**
- GET / - получение всех библиотек с пагинацией и сортировкой
- GET /{id} - получение библиотеки по ID
- GET /search - поиск библиотек по имени и источнику
- GET /healthy - фильтрация здоровых библиотек по порогу оценки
- GET /stats - общая статистика по библиотекам
- POST / - создание новой библиотеки
- PUT /{id} - обновление библиотеки
- DELETE /{id} - удаление библиотеки

**Лицензии (/api/v1/licenses)**
- GET / - получение всех лицензий с пагинацией
- GET /{id} - получение лицензии по ID
- GET /search - поиск лицензии по имени
- GET /type/{type} - фильтрация по типу (PERMISSIVE, COPYLEFT, PROPRIETARY)
- GET /osi-approved - получение OSI-одобренных лицензий
- POST / - создание новой лицензии
- PUT /{id} - обновление лицензии
- DELETE /{id} - удаление лицензии

**Задачи сканирования (/api/v1/scan-jobs)**
- GET / - получение всех задач с пагинацией
- GET /{id} - получение задачи по ID
- GET /status/{status} - фильтрация по статусу (PENDING, RUNNING, COMPLETED, FAILED)
- GET /source/{source} - фильтрация по источнику данных
- GET /recent - получение последних задач
- GET /statistics - статистика по статусам задач
- POST / - создание новой задачи
- PATCH /{id}/status - обновление статуса задачи
- DELETE /{id} - удаление задачи
